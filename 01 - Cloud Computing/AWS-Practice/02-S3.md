# AWS S3 — Simple Storage Service

## 1. Core Concepts

### What is S3?
S3 is AWS's **object storage** service — store any amount of data (objects) in containers called **buckets**. Objects can be up to 5 TB each. S3 provides 11 nines (99.999999999%) durability by replicating data across a minimum of 3 AZs.

### Storage Classes (Know These Cold!)
| Storage Class | Retrieval | Min Storage | Min Billable Size | Use Case |
|--------------|-----------|-------------|-------------------|----------|
| **S3 Standard** | Milliseconds | None | None | Frequently accessed data, active web content |
| **S3 Intelligent-Tiering** | Milliseconds | None | 128 KB | Unknown access patterns — auto-moves objects between tiers |
| **S3 Standard-IA** | Milliseconds | 30 days | 128 KB | Infrequently accessed, must be available immediately |
| **S3 One Zone-IA** | Milliseconds | 30 days | 128 KB | Re-creatable infrequently accessed data (no multi-AZ) |
| **S3 Glacier Instant** | Milliseconds | 90 days | 128 KB | Archive with rare but instant access (medical images) |
| **S3 Glacier Flexible** | 1–12 hours | 90 days | 40 KB | Long-term backup, retrievable in bulk (5–12 hr) or expedited (1–5 min) |
| **S3 Glacier Deep Archive** | 12–48 hours | 180 days | 40 KB | Cheapest storage (~$1/TB/month), regulatory archives, 7-10 yr retention |

### Bucket & Object Fundamentals
- **Bucket name**: Globally unique, lowercase, 3–63 characters, no IPs
- **Object key**: Full path within bucket (`images/2024/photo.jpg`)
- **Object size**: 0 bytes – 5 TB; use **Multipart Upload** for > 100 MB
- **Versioning**: Keeps all versions of every object — enables point-in-time recovery and protects against accidental delete
- **S3 Object Lock**: WORM (Write Once Read Many) — prevent deletion for a retention period (Compliance or Governance mode)

### Access Control
| Mechanism | Scope | Notes |
|-----------|-------|-------|
| **Bucket Policy** | Bucket + objects | JSON policy; most flexible; can grant cross-account access |
| **IAM Policy** | Users/Roles/Groups | Controls what principals can do on which buckets |
| **ACLs** | Objects | Legacy; AWS recommends disabling ACLs and using bucket policies |
| **Block Public Access** | Account/Bucket | Override all ACLs and policies — must be off to allow public access |
| **S3 Access Points** | Named entry points | Simplify access for large shared datasets (different teams, different prefixes) |

### Encryption
| Option | Who Manages Keys | Algorithm |
|--------|-----------------|-----------|
| **SSE-S3** | AWS | AES-256, S3-managed keys |
| **SSE-KMS** | AWS KMS (you control key policy) | AES-256, audit trail in CloudTrail |
| **SSE-C** | Customer | You provide key with each request; AWS doesn't store it |
| **Client-side** | Customer | Encrypt before upload; AWS never sees plaintext |

### Key Features
- **Lifecycle Policies**: Automatically transition objects between storage classes or expire them
- **Replication**:
  - *CRR (Cross-Region Replication)*: DR, compliance, latency reduction for global users
  - *SRR (Same-Region Replication)*: Log aggregation, live replication between production and test
- **S3 Transfer Acceleration**: Uses CloudFront edge network for faster long-distance uploads
- **Requester Pays**: Requester bears transfer and request costs (useful for public datasets)
- **Event Notifications**: Trigger Lambda, SQS, SNS on object create/delete/restore
- **S3 Select / Glacier Select**: Query CSV/JSON/Parquet objects with SQL — retrieve subset without downloading the whole object
- **S3 Inventory**: Report of all objects in a bucket (useful for audits)
- **Multipart Upload**: Split large objects; upload parts in parallel; required for > 5 GB

### Performance Best Practices
- **Prefix randomization**: S3 partitions by prefix — use different prefixes for high-throughput workloads (3,500 PUT + 5,500 GET per second per prefix)
- **S3 Transfer Acceleration**: For global upload scenarios
- **Byte-Range Fetches**: Parallelize large downloads by fetching ranges concurrently

### Pre-Signed URLs
Temporarily share private objects without making them public. Expire after a specified duration. Generated using AWS SDK or CLI:
```bash
aws s3 presign s3://my-bucket/my-file.pdf --expires-in 3600
```

## 2. Interview Questions & Answers

### Q1. What is the difference between S3 Standard-IA and S3 One Zone-IA?
Both are for infrequently accessed data. **Standard-IA** stores data across at least 3 AZs — same durability as Standard. **One Zone-IA** stores data in a single AZ — 20% cheaper but data is lost if that AZ fails. Use One Zone-IA only for data you can re-create (thumbnails, transcoded media, derived analytics).

---

### Q2. How does S3 Versioning work and why is it important?
When Versioning is enabled, S3 keeps every version of every object. Deleting an object adds a **delete marker** (the old versions remain). To permanently delete, you must delete each version explicitly. This protects against accidental deletes and enables point-in-time recovery. Once enabled, versioning can only be **suspended** (not disabled).

---

### Q3. What is S3 Object Lock and when would you use it?
Object Lock enforces **WORM** (Write Once Read Many) on objects. Two modes:
- **Governance**: Users with special IAM permission can override/delete
- **Compliance**: No one — not even root — can delete or modify until the retention period expires

Use cases: Financial records, healthcare data, legal documents, regulatory compliance (SEC Rule 17a-4, FINRA).

---

### Q4. Explain the difference between S3 bucket policies and IAM policies.
Both are JSON access control policies evaluated together.
- **Bucket Policy**: Attached to the bucket; controls which **principals** (users, roles, services, accounts) can access it. Enables **cross-account access**.
- **IAM Policy**: Attached to a user/role; controls what **resources** that principal can access across all services.

AWS evaluates both — access is granted only if **neither** denies and **at least one** allows.

---

### Q5. What is S3 Transfer Acceleration?
It routes uploads through AWS **CloudFront edge locations** closest to the uploader, then uses AWS's private backbone network to transfer to the S3 bucket's region. Reduces latency for global uploads by 50–500%. You get a distinct endpoint: `bucket.s3-accelerate.amazonaws.com`. You pay an additional per-GB fee.

---

### Q6. How does S3 replication work? What are the prerequisites?
**Prerequisites**: Versioning must be enabled on both source and destination buckets. The IAM role for S3 replication must have read permission on source and write on destination.  
**CRR** (Cross-Region): Reduces latency for global users, meets data sovereignty requirements, DR.  
**SRR** (Same-Region): Log aggregation, sync prod→test.  
**Important**: Replication only applies to objects written **after** replication is enabled. Existing objects must be replicated using **S3 Batch Operations**.

---

### Q7. What is S3 Intelligent-Tiering and when should you use it?
S3 Intelligent-Tiering monitors access patterns and automatically moves objects between Frequent Access, Infrequent Access, and Archive tiers. There's a small monitoring fee (~$0.0025/1,000 objects/month) but no retrieval fee and no transition fee. Best for data with **unknown or changing access patterns** — avoid it for small objects (<128 KB, as monitoring fee exceeds savings) or objects you know will always be frequently accessed.

---

### Q8. How would you secure a static website hosted on S3?
1. Enable **Block Public Access** at bucket level
2. Use **CloudFront** as the CDN in front of the bucket
3. Use a **CloudFront Origin Access Control (OAC)** — bucket policy allows only the CloudFront distribution, not the public
4. Enable **WAF** on the CloudFront distribution to filter malicious requests
5. Enable **S3 server access logging** and send to a separate logging bucket
6. Enable **SSE-S3** or **SSE-KMS** encryption at rest

---

### Q9. What is a Pre-Signed URL and what are its use cases?
A pre-signed URL is a time-limited URL that grants **temporary access** to a private S3 object to anyone who has the URL. Signed with the credentials of the IAM user/role that generated it.  
Use cases:
- Sharing a file for download without making the bucket public
- Allowing a mobile app to upload directly to S3 without routing through your backend (pre-signed PUT URL)
- Generating shareable report links that expire in 24 hours

---

### Q10. What are S3 Lifecycle Policies? Give a practical example.
Lifecycle policies automate moving objects between storage classes or deleting them based on age.  
**Example — Log Archive Policy:**
```
Day 0: Objects created in s3://my-logs/app-logs/
Day 30: Transition to S3 Standard-IA (access rare after a month)
Day 90: Transition to S3 Glacier Flexible (compliance archive)
Day 365: Delete (retention period expired)
```

---

### Q11. What is the maximum object size in S3 and how do you handle large files?
Maximum object size: **5 TB**. For objects > 100 MB, you should use **Multipart Upload**; it's required for > 5 GB:
- Split file into parts (5 MB – 5 GB each)
- Upload parts in parallel
- S3 assembles the object atomically

Benefits: Better throughput, resume on failure, upload while file is still being created.

---

### Q12. How does S3 achieve 11 nines of durability?
S3 automatically replicates data across **at least 3 Availability Zones** within a region. It performs regular data integrity checks and auto-heals corrupted data. The 99.999999999% figure means losing 1 object per 10 billion stored per year.

---

### Q13. What is S3 Select? How does it help?
S3 Select lets you run simple **SQL queries** against S3 objects (CSV, JSON, Parquet) and retrieve only the matching records — not the entire file. This can reduce data transfer by 400x and query time by 400%. Useful for large log files or datasets where you only need a slice.

---

### Q14. Explain S3 Event Notifications.
S3 can trigger notifications on object events (PUT, DELETE, COPY, restore). Destinations:
- **SQS**: Decouple downstream processing
- **SNS**: Fan out to multiple subscribers
- **Lambda**: Serverless processing (most common)
- **EventBridge**: Rich routing and filtering to many AWS services

Use case: New image uploaded → Lambda triggered → thumbnail generated → stored back in S3.

---

### Q15. What is S3 Access Points and when would you use it?
Access Points are named network endpoints attached to a bucket. Each access point has its own policy and can be restricted to a VPC. They simplify managing access to shared data in large organizations:
- **Data team**: Access Point with read on `/analytics/` prefix
- **App team**: Access Point with read/write on `/uploads/` prefix
- **Compliance team**: Read-only Access Point on `/audit/`

Instead of one complex bucket policy, you manage smaller focused policies per access point.

## 3. Real-World Use Case: Data Lake Architecture

### Scenario
A media company needs to:
- Ingest 500 GB of raw video/image assets daily from global studios
- Transform and catalog the data for analytics
- Archive raw masters after 90 days (compliance: 7-year retention)
- Allow machine learning team to query without downloading full files
- Prevent accidental deletion of master content

### Architecture

```
┌─────────────────────────────────────────────────────────────────┐
│                         INGESTION LAYER                         │
│                                                                 │
│  Global Studios ──► S3 Transfer Acceleration ──► S3 Bucket     │
│                      (edge → backbone)           (raw-zone)    │
│                                                                 │
│  S3 Event Notification ──► SQS ──► Lambda (thumbnail gen)      │
└──────────────────────────────────────┬──────────────────────────┘
                                       │ CRR (Cross-Region Replication)
                                       ▼
┌─────────────────────────────────────────────────────────────────┐
│                       STORAGE ZONES (S3)                        │
│                                                                 │
│  raw-zone/          ──► S3 Standard (0-90 days)                │
│  processed-zone/    ──► S3 Intelligent-Tiering                 │
│  archive-zone/      ──► S3 Glacier Deep Archive (90d+, 7yr)    │
│                                                                 │
│  All zones: Versioning ON, Object Lock (Compliance, 7yr)       │
│             SSE-KMS encryption, Block Public Access            │
└──────────────────────────────────────┬──────────────────────────┘
                                       │
┌──────────────────────────────────────▼──────────────────────────┐
│                       CATALOG & QUERY LAYER                     │
│                                                                 │
│  AWS Glue Data Catalog ──► Athena (SQL on S3)                  │
│                            S3 Select (object-level query)       │
│                                                                 │
│  Access Points:                                                 │
│    ml-team-ap  (processed-zone/, VPC-restricted)               │
│    analytics-ap (processed-zone/, read-only)                   │
│    ingest-ap   (raw-zone/, write-only for studios)             │
└─────────────────────────────────────────────────────────────────┘
```

### Lifecycle Policy (per bucket/prefix)

| Zone | 0–30 Days | 31–90 Days | 91 Days + |
|------|-----------|------------|-----------|
| raw-zone | Standard | Standard-IA | Glacier Deep Archive |
| processed-zone | Intelligent-Tiering | Intelligent-Tiering | Glacier Flexible |
| archive-zone | Glacier Deep Archive | — | — |

### Key Design Decisions

| Decision | Reason |
|----------|--------|
| S3 Transfer Acceleration | Studios in Tokyo, London, São Paulo — 60% faster global uploads |
| Versioning + Object Lock (Compliance) | Master content cannot be deleted by anyone — legal requirement |
| CRR to secondary region | DR — recover from region-level event |
| Access Points per team | Least-privilege access without complex bucket policies |
| S3 Select + Athena | ML team queries specific scenes from multi-GB files — saves $$ on data transfer |
| SSE-KMS | Audit trail via CloudTrail for every key access — satisfies compliance auditor |

### Interview Narration (White-board Script)
> "I'd design the data lake with three S3 zones: raw, processed, and archive. Global studios upload directly to S3 using Transfer Acceleration so we get fast reliable ingestion regardless of their geography. Lifecycle policies automatically move raw content to Glacier Deep Archive at 90 days with Object Lock in Compliance mode to satisfy the 7-year retention requirement. For access control I'd use S3 Access Points — each team gets a named endpoint with its own scoped policy rather than one massive bucket policy. The ML team uses Athena with S3 Select so they can query specific frames without downloading 10 GB video files."
